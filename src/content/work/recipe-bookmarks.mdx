## Background

This started as a practical problem: saving recipes from across the web in a way that's actually useful — with previews, categories, and search — without paying a monthly fee for a tool that does 80% of what's needed.

## The Problem

Generic bookmark tools don't understand recipes. There's no category structure, no ingredient context, and no admin layer for managing the library. Furthermore, links break over time, leading to lost recipes.

## What I Built

A full-stack Progressive Web App running on Cloudflare Pages and Workers:

- **Server-Side Open Graph Scraping:** A Cloudflare Worker automatically fetches the URL and parses OG tags and JSON-LD data to populate the title and image. It uses a mobile Safari User-Agent to bypass certain social media restrictions.
- **Automated Link Checking:** Two separate systems ensure links stay alive: a background check runs on admin login, and a weekly GitHub Actions cron job calls an edge endpoint to test all URLs, emailing a report via Resend if any links return a 405/404.
- **Built-in Full Text Search:** Uses SQLite's FTS5 virtual table on Cloudflare D1. The database auto-syncs the `recipes_fts` table via triggers, providing fast keyword search across titles and notes without an external search service.
- **PWA Web Share Target:** On Android, the app integrates directly with the OS share-sheet. Sharing a link from a browser sends it directly to the app's `ShareReceiver` component, parsing the URL and opening the Add Recipe form automatically.
- **Custom Auth:** Implemented a lightweight auth system using PBKDF2 password hashing and HTTP-only JWT session cookies, with distinct user and admin roles.

## Key Technical Decision

<Callout>
**Building search into the database instead of adding a third-party service.**
Using SQLite's built-in FTS5 full-text search kept costs at zero and eliminated an external dependency — at the cost of less sophisticated relevance ranking. For a personal recipe app, that tradeoff is worth it, keeping the entire application self-contained on Cloudflare's edge infrastructure.
</Callout>
