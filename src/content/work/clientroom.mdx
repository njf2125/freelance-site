## Background

Creatives and event vendors working one-on-one with clients need a simple way to keep everything organized. But most project management tools are too complex, and email threads quickly become a mess.

## The Problem

When working with clients, communication happens over email, files are shared via Google Drive or Dropbox, and invoices are sent through Stripe or Quickbooks. The client has to keep track of multiple links and systems, which leads to confusion and delays. Moreover, asking clients to create yet another account just to view their project is a point of friction.

## What I Built

A single room for each client. Everything related to their project lives in one place, accessible via a private, secure link. 

- **Two Distinct Flows:** Freelancers authenticate via Firebase Auth (email/password or Google OAuth) to manage their rooms from a dashboard. Clients, on the other hand, are completely accountless. They access their specific room via a unique 24-character token URL.
- **Real-time Messaging:** Built-in chat using Firestore's realtime capabilities (`onSnapshot`) synced directly to the React Query cache, ensuring all communication stays within the context of the project.
- **File Sharing & Milestones:** Securely share deliverables, contracts, and assets. Clear timelines so clients always know what happens next.
- **Invoices:** Integrated payments handled securely via Stripe.
- **Editorial Aesthetic:** Designed specifically for creatives. The UI features a warm, editorial look using serif fonts (Cormorant Garamond) for headings and sans-serif (Jost) for body text. It deliberately avoids rounded corners for a sharp, sophisticated feel, complemented by a subtle noise texture overlay.

## Key Technical Decision

<Callout>
**Accountless Client Access and Single State Layer.**
To remove friction, clients never create an account; their access is token-gated at the data level and stored in `sessionStorage`. For the architecture, I chose to use TanStack React Query as the *only* state management layer—eschewing Context or Redux—to seamlessly sync server state and Firestore realtime listeners into the UI.
</Callout>
